# dots.tts 整合包使用说明

> 2B 全连续自回归 TTS · 48kHz · 零样本声音克隆
> 本整合包基于内置的 `python-3.11.9`（Windows 嵌入式 Python），**解压即用，无需安装 conda/Python**。

---

## 一、快速开始

1. 把整个文件夹解压到一个**英文路径、无空格**的目录（例如 `D:\dots.tts`）。
2. 确认是 **NVIDIA 显卡**且驱动较新。
   - **不需要安装 CUDA / CUDA Toolkit**：CUDA 运行库已随整合包一起带好（在 `torch\lib`）。
   - **只需装显卡驱动**：命令行运行 `nvidia-smi`，右上角 `CUDA Version` 显示 **12.x（≥12.0）** 即可；
     嫌麻烦就直接更新到最新 NVIDIA 驱动。A 卡 / 核显 / 无独显**无法运行**。
3. 双击 **`启动WebUI-soar.bat`**（音质 / 克隆相似度最佳）或 **`启动WebUI-mf.bat`**（更快、更省显存）。
4. 模型**已经打包在整合包里**（`hf_cache\`），**无需联网下载，离线也能跑**。
   首次启动会做一次预热/编译（约 1–2 分钟，黑窗口里有日志），**请不要关闭窗口**；之后启动很快。
5. 加载完成后会**自动打开浏览器**（soar 是 `http://127.0.0.1:7861`，mf 是 `7860`），即可使用。

> 模型就在整合包内的 `hf_cache\` 目录里，整个文件夹拷到别的（有 NVIDIA 显卡的）机器即可继续用。

---

## 二、怎么用（声音克隆）

默认模型是声音克隆模型，**需要一段参考音频**来决定音色：

1. 在左侧「参考音频 / Prompt Audio」上传一段 **约 10 秒** 的清晰人声 wav。
2. 在「参考音频转写 / Prompt Text」里填上**这段音频实际说的文字**（要对得上，否则音质/稳定性下降）。
3. 在右侧「待合成文本 / Text」输入你想要合成的内容（中英文均可）。
4. 点 **Generate**，稍候即可在下方播放/下载生成的音频。

### 参考音频小贴士
- 参考音频保持 ~10s 即可，更长不会更好。
- 转写文本要和音频内容一致。
- 参考音频越干净（高采样率、无底噪、无尾音）克隆效果越好。
- 想要不同的语气/节奏，改一下 Settings 里的 **Seed** 多试几次。
- 多音字可用拼音强制读音，例如把「好」写成带声调拼音 `hào`：`我生平不hào此道`
  （只认带声调拼音 `hǎo/hào`，不认 `hao4`）。

---

## 三、参数说明（右侧 ⚙️ Settings）

| 参数 | 说明 | 建议 |
|---|---|---|
| Num Steps | 采样步数，越大越精细越慢 | MF 模型用 **4** 即可 |
| Guidance Scale | CFG 强度 | MF 模型已把 CFG 蒸馏进模型，调它基本无效，保持默认 |
| Seed | 随机种子，固定种子结果可复现 | 想换语气就换个数 |
| Normalize Text | 文本归一化 | **本整合包不可用**（见下方说明），保持不勾选 |

## 四、命令行用法（可选）

整合包也注册了命令行。用内置 Python 调用：

```bat
python-3.11.9\python.exe -m dots_tts.cli --model-name-or-path rednote-hilab/dots.tts-mf --text "你好，这是一段测试。" --prompt-audio 参考.wav --prompt-text "参考音频的文字" --num-steps 4 --output out.wav
```

（命令行同样需要先设置镜像：`set HF_ENDPOINT=https://hf-mirror.com`，或直接在 `启动WebUI.bat` 同款环境下运行。）

---

## 五、两个画质档怎么选

整合包内置了两个模型，**对应两个启动脚本，直接双击即可,不用改任何配置**：

- **`启动WebUI-mf.bat`** → `dots.tts-mf`：最省显存、最快，适配 8GB 显存笔记本（端口 7860）。
- **`启动WebUI-soar.bat`** → `dots.tts-soar`：音质/克隆相似度最佳，多步采样更慢更吃显存（端口 7861）。

8GB 显存追求质量就用 soar，偏紧时把 Num Steps 调小；要快就用 mf。

## 六、进阶：接入本地大模型，实时语音对话（双工流式）

整合包附带一个示例脚本 `scripts\demo_ollama_duplex.py`，演示**「本地大模型边生成文字、TTS 边把它读出来」**的双工流式效果（适合录视频 / 做语音助手 Demo）。

### 前置条件
1. 安装并启动 **[Ollama](https://ollama.com)**，拉一个对话模型。**推荐用小模型**（更流畅）：
   ```
   ollama pull qwen2.5:1.5b
   ```
2. 安装实时播放依赖（用整合包内置 Python）：
   ```bat
   python-3.11.9\python.exe -m pip install sounddevice -i https://pypi.tuna.tsinghua.edu.cn/simple
   ```

### 方式 A：实时语音对话（推荐，**双击 `实时语音对话.bat`**）
这是最接近「和模型语音聊天」的玩法：**这个窗口就是聊天框**——你打字提问，大模型的回答
一边在屏幕上逐字滚动，一边由 dots.tts **实时读出来**。输入 `/quit` 退出。

> 为什么不是去「监听」另一个 `ollama run` 窗口？因为 Windows 上无法在保持交互的同时
> 旁路截获另一个程序的输出（管道会让回答从你的窗口里消失、还会破坏 REPL）。所以这个脚本
> 干脆**取代 `ollama run`**——打字体验完全一样，只是多了实时语音。

手动运行等价命令（在 `启动WebUI.bat` 同款环境下）：
```bat
python-3.11.9\python.exe scripts\ollama_voice_chat.py --ollama-model qwen2.5:1.5b --prompt-audio 参考.wav --prebuffer 1.5
```
- `--prompt-audio` 指定参考音频决定**音色**（双工模式只看音频、不需要转写文本）。
- `--prebuffer` 起播前先攒多少秒音频（越大越不容易断，但起播更慢）。
- 脚本自带「语音助手」系统提示，会让模型**回答简短、口语化、不带 Markdown**，更适合朗读。
- 加 `--no-play` 则只显示文字不出声；`--system-prompt "..."` 可自定义模型人设。

### 方式 B：单次提问、生成成 wav（适合录视频，画面零卡顿）
```bat
python-3.11.9\python.exe scripts\demo_ollama_duplex.py --ollama-model qwen2.5:1.5b --prompt-audio 参考.wav --play
```
- 只问一个问题（`--prompt "你的问题"` 自定义），不带 `--play` 则只生成 `ollama_duplex.wav`。
- 也可直接双击 **`录视频Demo.bat`**：输入问题→终端文字实时滚动→生成完自动播放干净音频。

### 方式 C：用云端 DeepSeek（不想装 Ollama 时用）
**双击 `实时语音对话-DeepSeek.bat`**：用云端 DeepSeek 大模型替代本地 Ollama，玩法与方式 A 完全一样。
- 需要**联网** + 一把你自己的 **DeepSeek API Key**（到 https://platform.deepseek.com 注册获取）。
- 用记事本打开该 `.bat`，把 `set "DEEPSEEK_API_KEY=sk-your-key-here"` 的占位符换成你的真实 Key。
- 好处：本地不跑大模型，只有 TTS 占显存，往往**更流畅**。

### 关于卡顿（重要，先看这里）
本整合包**已内置 Triton**，所以 **`--optimize`（torch.compile 加速）可以用**——实时语音对话的两个脚本
（`实时语音对话.bat` / `实时语音对话-DeepSeek.bat`）默认已带上。
开启后**首次会编译**（约 1–2 分钟，属正常预热）；编译完成后 TTS 明显变快，多数显卡上能接近甚至达到实时，
配合缓冲基本可做到**边生成边播、不卡**。
两点注意：① 编译结果按机器/显卡缓存，**每台新机器首次都要再等一次编译**；② 若 TTS 与大模型抢同一块显卡，速度仍会下降。
个别环境若编译失败，用下面"只生成 wav"的方式兜底即可。
（想让 WebUI 也更快？用记事本在 `启动WebUI-*.bat` 里 `app.py ...` 那一行末尾加上 ` --optimize`，同样首次会编译。）

按你的目的二选一：

- **要画面绝对不卡（推荐录视频用）**：**不要加 `--play`**，让脚本只生成 wav。
  终端里文字仍是实时逐字滚动，录完再播放生成好的 `ollama_duplex.wav`，**保证零卡顿**。
- **要现场出声的"近实时"效果**：加 `--play`，并把缓冲调大，例如 `--prebuffer 2.5`。
  句子越短越顺；第一次会先攒 2~3 秒再开始播。

| 现象 | 原因 | 处理 |
|---|---|---|
| 开头等一两秒才出声 | 首次解码在预热（已自带 warmup） | 正常 |
| 播放断断续续 | 未开 `--optimize`（或仍在首次编译/预热），或与 LLM 抢显存 | 确认带了 `--optimize` 并等编译完成；加大 `--prebuffer`；句子说短点；或改用"只生成 wav 再播" |
| 整体偏慢 | LLM 和 TTS 挤同一块显卡 | 换小 LLM（`qwen2.5:1.5b`）；两张卡时让 LLM 和 TTS 各用一张 |

> 仍不顺：确认 `--optimize` 已生效、首次编译已完成；把 `--prebuffer` 调大、句子说短；给 TTS 单独一块显卡；或直接用"只生成 wav 再播"（录视频零卡顿）。

## 七、常见问题

- **黑窗口卡很久没反应？** 模型已内置、不联网下载，所以首次的等待主要是 **torch.compile 预热编译**
  （约 1–2 分钟，看到窗口里日志在滚动就是正常）。编译完成后才会弹出浏览器；之后启动会快很多。
- **提示显存不足 / CUDA out of memory？** 关掉占用显存的其它程序；或按第五节换 soar 时反而更吃显存，建议保持默认 mf；也可在 Settings 里把 Num Steps 调小。
- **端口 7860 被占用？** 编辑 `启动WebUI.bat`，把 `--port 7860` 改成别的端口（如 7861）。
- **浏览器没自动打开？** 手动访问 `http://127.0.0.1:7860`。
- **想离线/重装依赖？** 双击 `安装环境.bat` 可重建 Python 环境（需要联网）。
- **`Normalize Text` 为什么不可用？** 该功能依赖 `WeTextProcessing`(pynini)，
  在 Windows 嵌入式 Python 上无法安装，故整合包默认禁用；其余功能不受影响。

## 八、整合包是怎么做出来的（给维护者）

1. 内置 `python-3.11.9` 为 Windows 嵌入式包，已修改 `python311._pth` 启用 `import site`
   并把 `..\src`、`..`（仓库根）加入搜索路径。
2. `安装环境.bat` 依次：装 pip → 装 CUDA 12.8 版 torch/torchaudio → 装
   `pack\requirements_pack.txt` 里的其余依赖（**不含 WeTextProcessing**）→ `pip install -e . --no-deps`。
3. `src/dots_tts/utils/text.py` 已把 `lingua` 和 `tn`(WeTextProcessing) 改为**懒加载**，
   这样不装 pynini 也能 `import` 整个 runtime。
4. `启动WebUI.bat` 设置 `HF_ENDPOINT` 镜像、`HF_HOME` 本地缓存、
   `DOTS_TTS_OPEN_BROWSER=1` 自动开浏览器，并以 `dots.tts-mf` 启动 Gradio。
